# 项目任务与临时事项管理工具

一款轻量级的静态网页工具，用于管理项目待办任务和临时事项，支持智能延期计算、文档附件归档和数据导入导出。

## 功能特性

### 核心功能

- **任务管理**：按项目维度管理待办任务，支持状态跟踪、实时搜索、计划调整
- **临时事项**：记录会议、紧急任务等临时事项，支持附件上传
- **延期计算**：自动计算临时事项对任务的影响，支持计划调整分段计算
- **文档附件**：基于 File System Access API，附件自动归档到本地指定路径
- **数据导入导出**：支持 CSV/JSON 格式的批量导入和数据备份
- **操作反馈**：所有增删改操作均有 Toast 提示，无需依赖弹窗确认
- **键盘快捷键**：Escape 关闭弹窗

### 延期计算规则

公式：`offset = 1 + floor((hours - 2) / 8)`（当 hours > 2 时），前 2 小时计 1 个工作日，之后每 8 小时加 1 个工作日。

| 累计时长 | 延期工作日 | 计算方式 |
|---------|-----------|---------|
| ≤ 2小时 | 1天 | 固定 1 天 |
| 10小时 | 2天 | 1 + floor((10-2)/8) = 1 + 1 = 2 |
| 18小时 | 3天 | 1 + floor((18-2)/8) = 1 + 2 = 3 |
| 26小时 | 4天 | 1 + floor((26-2)/8) = 1 + 3 = 4 |

- 自动跳过周末和法定节假日
- 支持调休工作日识别
- 节假日数据自动更新（来源：[holiday-cn](https://github.com/NateScarlet/holiday-cn)）

## 快速开始

### 直接使用

1. 下载 `index.html` 文件
2. 双击用浏览器打开
3. 开始使用

### 部署到 Web 服务器

将整个 `task-manager` 目录部署到任意 Web 服务器即可。

## 使用说明

### 任务管理

1. 点击「新建任务」添加任务
2. 填写项目号、任务名称、时间等信息
3. 在列表中查看任务状态和延期情况
4. 使用搜索框按项目号或任务名称快速筛选

### 计划调整

1. 点击任务表格最后一列的「延期」按钮
2. 在弹出的延期时间线中点击「计划调整」
3. 设定新的目标完成日期
4. 系统会自动以新日期为基准重新分段计算后续延期

### 创建临时事项

1. 切换到「临时事项」标签
2. 点击「新建临时事项」
3. 填写事项日期、时长（小时/天）
4. 选择影响范围：
   - **所有未完成任务**：自动影响所有未完成任务
   - **指定任务**：手动选择受影响的任务
5. （可选）拖拽或点击选择附件文件，保存后自动存入本地存档

> 延期时长由系统根据「时长」自动计算，无需额外填写。附件功能需先在「数据管理」中设置存档路径。

### 查看延期影响

1. 切换到「审阅」标签
2. 点击任务的「延期记录」链接
3. 查看影响该任务的所有临时事项

### 数据导入

#### CSV 格式

```csv
项目号,任务名称,开始时间,计划完成时间,状态
PRJ-001,需求分析,2024-06-01,2024-06-15,未开始
PRJ-001,前端开发,2024-06-10,2024-06-25,进行中
```

#### JSON 格式

```json
[
  {
    "projectName": "PRJ-001",
    "taskName": "需求分析",
    "startDate": "2024-06-01",
    "plannedEndDate": "2024-06-15",
    "status": "未开始"
  }
]
```

#### 导入模式

导入 JSON 备份数据时，系统会提示选择导入模式：

- **覆盖导入**（点击「确定」）：清除当前所有数据，替换为导入的数据
- **合并导入**（点击「取消」）：保留当前数据，将导入的数据中 ID 不重复的部分追加进来

## 技术特点

- **零依赖**：纯原生 HTML/CSS/JavaScript，无任何第三方库
- **单文件部署**：整个应用在一个 HTML 文件中
- **离线可用**：数据存储在浏览器本地，无需网络
- **自动更新**：节假日数据从 CDN 动态加载

## 数据备份

### 文档附件存档

1. 切换到「数据管理」标签
2. 在「文档附件存档」卡片中点击「设置路径」
3. 选择本地文件夹作为存档根目录
4. 之后创建临时事项时即可添加附件，自动归档到 `{日期}_{ID}/` 子文件夹
5. 更换存档路径后，需手动将旧路径下的文件迁移到新路径

### 导出数据

1. 切换到「数据管理」标签
2. 点击「导出数据 (JSON)」
3. 保存下载的 JSON 文件

### 导入数据

1. 切换到「数据管理」标签
2. 点击「导入数据 (JSON)」
3. 选择之前导出的 JSON 文件

## 常见问题

### Q: 数据会丢失吗？

A: 数据存储在浏览器 localStorage 中，清除浏览器数据会导致丢失。建议定期导出备份。

### Q: 附件存在哪里？

A: 附件文件存储在用户指定的本地文件夹（通过「数据管理」→「文档附件存档」设置）。每次新建临时事项保存附件时，系统会在存档目录下自动创建 `{日期}_{事项ID}/` 子文件夹存放文件。更换存档路径后旧附件需手动迁移。

### Q: 附件需要什么浏览器？

A: 需要使用支持 File System Access API 的浏览器（Chrome 86+ / Edge 86+）。Firefox 暂不完全支持。

### Q: 能多人使用吗？

A: 当前版本为单机版，数据存储在本地浏览器。如需多人协作，需要接入后端服务。

### Q: 节假日数据准确吗？

A: 节假日数据来源于 [NateScarlet/holiday-cn](https://github.com/NateScarlet/holiday-cn)，自动从国务院公告抓取。2024-2026年数据准确，2027年以后为预估数据。

### Q: 如何修改延期计算规则？

A: 修改 `index.html` 中的 `hoursToWorkingDaysOffset` 函数即可。
